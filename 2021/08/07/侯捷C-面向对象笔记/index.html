头文件的正规写法

防卫式声明（guard）

编译器对头文件进行以下预处理：

  1. 编译器将处理掉所有注释，以空格代替；
  2. 删除#define，展开所有宏定义；
  3. 处理条件编译指令#if、#ifdef、#elif、#else、#endif；
  4. 处理#include，展开被包含的头文件（直接将头文件复制进文件）
  5. 保留编译器需要使用的#progma指令等等。

从第四点可知，如果不对头文件进行防卫式声明，那么可能会对某一个头文件进行多次处理，势必会产生重复定义和声明，也会让文件的大小变得很庞大（因为多次将头文件内容进行复制）。

六 参数传递：pass by value vs. pass by reference(to const)

pass by value就是把整个变量都压入栈中，若是这个变量超级大，那就不好。

所以如果传引用（pass by reference）的话，就是传这个变量在内存中的地址，传引用就相当于传指针，所以最好都是传引用。
